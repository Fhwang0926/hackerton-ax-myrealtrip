# 알려진 한계와 운영 참고사항

## 데이터 한계

- 인스타그램과 페이스북 게시글 단위 데이터는 기본적으로 샘플을 사용합니다. 공개 프로필 주소는 기록하지만, 안정적인 게시글 수집에는 소유 계정 내보내기 파일이나 공식 API 권한이 필요합니다.
- 네이버 블로그, 뉴스, 환율, 날씨는 생성된 리포트에서 실데이터로 수집됩니다.
- 휴일과 항공권 가격은 샘플을 사용합니다. 운영 환경에서는 실제 휴일 캘린더와 항공권 가격 데이터를 연결하는 것이 좋습니다.

## 실행 참고사항

- 공개 데이터 수집 요청은 로컬 Python 인증서 저장소 차이로 실행이 막히지 않도록 인증서 검증을 비활성화합니다.
- 원본 데이터는 타임스탬프가 붙은 파일로 저장하며 덮어쓰지 않습니다. 반복 실행하면 `data/raw/*` 스냅샷이 여러 개 생깁니다.

## 제출 참고사항

- `submission.zip`에는 플러그인 소스, 생성 리포트, 샘플, 테스트, `logs/chat.txt`가 포함됩니다.
- 현재 저장소에서 확인된 로그 원본은 `logs/chat.txt`입니다. 해커톤 제출처가 `chatgpt.md`, `codex.jsonl`, `claude.txt` 같은 별도 원본 로그를 요구하면 최종 제출 전에 편집하지 않은 원본을 추가해야 합니다.
- 프론트엔드 빌드나 Node.js 기반 UI 테스트는 실행하지 않았습니다. 리포트는 빌드가 필요 없는 정적 HTML/CSS/JS입니다.
